## 1. 系统简介

**T-Malls** 是基于 Cloudflare Workers + D1 Serverless SQL 构建的轻量级商城评论展示系统。系统核心能力包括：

- 基于 Cloudflare Workers 的边缘无服务器运行时，支持全球低延迟访问；
- 通过 D1 数据库存储评论（comments）信息，具备结构化 SQL 查询能力；
- 提供 HTML 页面渲染，用户访问即可看到数据库中最新的评论数据；
- 代码精简、部署便捷，适合作为电商评论模块的基础骨架。

系统入口文件位于 `src/index.ts`，页面渲染逻辑位于 `src/renderHtml.ts`，数据表结构位于 `migrations/0001_create_comments_table.sql`。

## 2. 系统架构

T-Malls 采用"边缘计算 + Serverless SQL"的极简分层架构：

```
┌──────────────────────────────────────────────────────────┐
│                   用户浏览器 (Browser)                    │
│               访问 https://<worker>.workers.dev          │
└──────────────────────────┬───────────────────────────────┘
                           │ HTTPS
                           ▼
┌──────────────────────────────────────────────────────────┐
│            Cloudflare Workers 边缘运行时                  │
│  ┌────────────────────────────────────────────────────┐  │
│  │  src/index.ts         ← 请求入口（fetch handler）   │  │
│  │  src/renderHtml.ts    ← HTML 渲染                   │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────┬───────────────────────────────┘
                           │ D1 Binding (env.DB)
                           ▼
┌──────────────────────────────────────────────────────────┐
│        Cloudflare D1 数据库 (d1-template-database)        │
│     表结构：comments (id, author, content)                │
└──────────────────────────────────────────────────────────┘
```

**架构概览图：**

![系统架构预览](https://imagedelivery.net/wSMYJvS3Xw-n339CbDyDIA/cb7cb0a9-6102-4822-633c-b76b7bb25900/public)

**关键组件说明：**

| 组件 | 作用 | 对应代码 |
| --- | --- | --- |
| Worker Fetch 入口 | 接收 HTTP 请求、查询数据库、返回 HTML | `src/index.ts` |
| HTML 渲染模块 | 将查询结果拼接为页面 HTML | `src/renderHtml.ts` |
| D1 数据库绑定 | 通过 `env.DB` 访问 D1 | `wrangler.json` |
| 迁移脚本 | 创建表结构并写入初始数据 | `migrations/0001_*.sql` |

## 4. 安装与部署

### 4.1 获取代码

```bash
git clone https://github.com/babbaabbbaaa/t-malls.git
cd t-malls
```

### 4.2 安装依赖

```bash
npm install
```

### 4.3 创建 D1 数据库

```bash
npx wrangler d1 create d1-template-database
```

执行成功后，终端会输出新数据库的 `database_id`，请将其复制并更新至 `wrangler.json` 中对应位置：

```json
"d1_databases": [
  {
    "binding": "DB",
    "database_id": "<这里粘贴你的 database_id>",
    "database_name": "d1-template-database"
  }
]
```

### 4.4 部署至 Cloudflare

```bash
npx wrangler deploy
```

部署完成后，终端会输出访问地址，例如：

```
https://t-malls.<your-subdomain>.workers.dev
```

## 5. 数据库初始化

项目 `migrations` 目录存放 SQL 迁移脚本。首次部署须执行：

```bash
# 远程数据库（生产）
npx wrangler d1 migrations apply DB --remote

# 本地数据库（调试）
npx wrangler d1 migrations apply DB --local
```

迁移脚本会自动创建 `comments` 表并写入 3 条示例数据：

```sql
CREATE TABLE IF NOT EXISTS comments (
    id INTEGER PRIMARY KEY NOT NULL,
    author TEXT NOT NULL,
    content TEXT NOT NULL
);

INSERT INTO comments (author, content) VALUES
    ('Kristian', 'Congrats!'),
    ('Serena',   'Great job!'),
    ('Max',      'Keep up the good work!');
```

## 7. 主要流程功能说明

当前版本已完成的主要流程如下：

### 7.1 请求处理流程

```
用户访问 URL
      │
      ▼
Worker.fetch() 被触发    (src/index.ts:4)
      │
      ▼
env.DB.prepare("SELECT * FROM comments LIMIT 3")
      │
      ▼
stmt.all() 执行查询，得到 results
      │
      ▼
renderHtml(JSON.stringify(results, 2))   (src/renderHtml.ts:1)
      │
      ▼
返回 HTML 响应（Content-Type: text/html）
```

## 10. 常见问题 FAQ

**Q1：部署后页面提示数据库连接失败？**
请确认：
1. `wrangler.json` 中的 `database_id` 已替换为 `wrangler d1 create` 返回的真实 ID；
2. 已执行 `npx wrangler d1 migrations apply DB --remote` 成功创建表结构。

**Q2：本地运行报错找不到 `DB` 绑定？**
请执行 `npm run seedLocalD1`（本地迁移）或直接运行 `npm run dev`，系统会自动完成本地数据库初始化。

**Q3：如何修改展示条数？**
修改 `src/index.ts` 中的 SQL：

```ts
const stmt = env.DB.prepare("SELECT * FROM comments LIMIT 10");
```

**Q4：如何新增一条评论？**
使用 wrangler CLI 执行 SQL：

```bash
npx wrangler d1 execute DB --remote \
  --command "INSERT INTO comments (author, content) VALUES ('Alice', 'Nice!')"
```

**Q5：如何查看运行指标？**
登录 Cloudflare Dashboard → Workers & Pages → 选择 `t-malls` → Observability 面板。
